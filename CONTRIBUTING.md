# Contributing

Thanks for your interest in contributing to the Document Approval System!

Please follow these steps:

- Fork the repository and create a feature branch from `main`.
- Keep changes small and focused; one feature or fix per PR.
- Write clear commit messages and PR descriptions.
- Run `npm ci` and `npm run lint` locally before opening a PR.

Testing & CI

- CI runs build and lint checks on push and PRs. Fix any issues reported by the workflow.

Code style

- This project uses TypeScript and Tailwind. Follow existing patterns when possible.

Reporting bugs

- Open an issue with the `Bug report` template and include steps to reproduce.
